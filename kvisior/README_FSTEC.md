# kvisior (с вкладкой «ФСТЭК 118» и колонкой БДУ)

Этот архив — **полная версия** kvisior UI с уже применёнными изменениями
для поддержки Приказа ФСТЭК № 118 и отображения BDU-ID в таблице уязвимостей.

## Что нового

### Страница Compliance

Появилась пятая вкладка **«ФСТЭК 118»** рядом с CIS / NIST / PCI / HIPAA.
22 чека по пунктам приказа, разбитые на 6 категорий:

| Категория     | Пункты приказа | Количество чеков |
|---------------|---------------|:-:|
| Изоляция      | п.9            | 3 |
| Уязвимости    | п.10           | 4 |
| Конфигурация  | п.11           | 9 |
| Целостность   | п.12           | 3 |
| Аудит         | п.13           | 3 |
| Доступ        | п.14           | 1 (помечен «в разработке») |

Каждый чек:
- ссылается на номер пункта приказа (`fstecRef`)
- помечен классом защиты, к которому относится (6/5/4)
- использует данные из уже существующих источников: snapshot sensor-а +
  результаты scanner-agent-а

### Страница Vulnerability Management

В таблице CVE добавлена колонка **«БДУ»** — идентификатор из банка данных
угроз ФСТЭК (если CVE в БДУ найден). Также:
- BDU-ID попал в CSV-экспорт
- Поиск по таблице теперь работает и по BDU-ID (например, `BDU:2024-01234`)

## Требования к backend

Чтобы колонка «БДУ» и чек `fstec-10.4` («Покрытие БДУ») работали,
**обязательно** нужно задеплоить scanner-agent с ФСТЭК-патчем
(см. соседний архив `scanner-agent-full.zip`).

До обновления scanner-agent:
- Вкладка «ФСТЭК 118» работает (использует snapshot sensor-а — этого
  достаточно для пунктов 9, 11, 14)
- Чеки 10.2, 10.3 работают (используют стандартный CVE summary)
- Чек 10.4 «Покрытие БДУ» будет показывать 0/1 (модуль не видит BDU-ID)
- Колонка «БДУ» в Vulnerabilities будет пустой (прочерки во всех строках)

После обновления scanner-agent — всё начинает работать без пересборки UI,
потому что поле `bduId` просто добавляется в JSON, который UI уже умеет
читать.

## Проверка после деплоя

1. Откройте `/compliance`
2. В строке переключения стандартов должна появиться пятая карточка
   «ФСТЭК 118» (красного цвета — `var(--danger)`)
3. Клик по ней — отображается список из 22 чеков с их статусом
4. Откройте `/vuln` → раздел CVEs
5. В таблице между «CVE ID» и «Severity» должна быть колонка «БДУ»
6. После нового скана часть строк получит BDU-ID (красные бейджи типа
   `BDU:2024-01234`)

## Известные ограничения

- Чек «14 — ролевая модель» намеренно показывает 0% — ролевой системы
  в UI пока нет, это честный gap, который нельзя скрыть
- Чек «12.3 — цифровая подпись образов» работает только если
  scanner-agent заполняет поле `signed` / `signature.verified` в
  результатах (по умолчанию — нет; требует интеграции с Cosign в
  отдельном provenance-verifier)
- БДУ ФСТЭК содержит подмножество CVE. У многих современных уязвимостей
  BDU-ID отсутствует — это нормально, в UI они показываются прочерком
